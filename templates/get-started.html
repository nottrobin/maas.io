Get started

MAAS is a bare-metal provisioning tool built for the cloud age. It is open source and free to use. Follow the instructions below to download, configure and install MAAS.

Talk to our experts

MAAS is supported by the cloud engineering experts at Canonical, the company behind Ubuntu. Contact us for more information or to discuss your specific needs.

Contact us  ›

1

Setup your hardware

You need one small server for MAAS, and at least one server with a BMC that you will manager. MAAS will provide DHCP and DNS on the network. We recommend you put it these two machines on their own switch, with a router.

2

Install Ubuntu Server

Download Ubuntu Server LTS and follow the step-by-step installation instructions on your MAAS server.

3

Install MAAS

From the command line type the commands below. Follow the step-by-step instructions to update relevant packages and install MAAS.

4

Create admin user

Create your admin credentials by typing.

Login to the MAAS UI at http://<maas.ip>/MAAS/

5

Turn on DHCP and DNS

Go to the “Clusters” tab, open the “Cluster master” link, hover over the row for the interface that is connected to the private network and select the edit icon.

  1. Set this interface to manage DHCP and DNS
  2. Set the “Router IP” to the default gateway
  3. Fill in details for the dynamic and static ranges.
6

Import images

Go to the “Images” tab and choose the OS releases and architectures you want to install dynamically.

7

Enlist and commission servers

Now you need to enlist and commission machines.

  1. PXE boot each machine once.
  2. You should see these machines appear in MAAS.
  3. You might need to set machines to PXE boot every time.
  4. Alternatively, add machines or chassis by selecting the add hardware button, and providing the BMC details.
  5. Select all the machines and “Commission” them using the “Take action” button.
  6. Wait until all machines have a “Ready” status and start deploying
